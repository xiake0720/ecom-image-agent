# Database Schema V1

This document describes the final v1 schema after the phase 2 schema closure. It is aligned with:

- Alembic head: `20260420_04`
- SQLAlchemy ORM models under `backend/db/models/`
- SQL reference: `docs/database/ecom_image_schema_v1.sql`

## Final Schema Decisions

- Final source of truth: current running implementation, with a minimal closing migration for drift.
- PostgreSQL target: PostgreSQL 16.
- UUID generation: application-side UUIDs. `pgcrypto` and `gen_random_uuid()` are not required by v1 migrations.
- `users.status`: `active`, `disabled`, `suspended`. The old document value `pending_verification` is removed.
- `idempotency_keys`: keep `request_key`, `request_hash`, `endpoint`, and `UNIQUE (user_id, request_key)`. No task-level `idem_key` column is added.
- `tasks.queued_at`: added as nullable operational timestamp. Existing queued/running/terminal rows are backfilled from `created_at` by migration `20260420_04`.
- `task_assets.role` and `task_assets.source_type`: kept as non-empty strings instead of hard database enums, because legacy main/detail flows already produce multiple role/source labels. Current dictionaries are documented below.
- JSON metadata columns remain named `metadata` in PostgreSQL. ORM attributes use `metadata_json` where needed to avoid SQLAlchemy's reserved `metadata` attribute.
- `image_edits`: the official v1 shape is the current implemented shape: `source_result_id`, `edit_task_id`, `edited_result_id`, `selection_type`, `selection`, `mode`, `metadata`.
- GIN indexes: added for JSONB columns that are operationally useful for filtering/debugging. No GIN index is added to `idempotency_keys.response_body` because it is response cache data, not a query surface.

## PostgreSQL Verification

Run against a dedicated PostgreSQL 16 test database:

```bash
set ECOM_TEST_POSTGRES_URL=postgresql+asyncpg://postgres:postgres@127.0.0.1:5432/ecom_image_agent_test
python scripts/verify_postgres_schema.py --reset-schema
```

The script executes:

- `alembic upgrade head`
- minimal CRUD over `users`, `idempotency_keys`, `audit_logs`, `tasks`, `task_results`, `task_assets`, and `image_edits`
- `alembic downgrade -1`
- `alembic upgrade head`

The integration test `tests/integration/test_postgres_schema_closure.py` runs the same verification when `ECOM_TEST_POSTGRES_URL` is set. Without that variable it is skipped to avoid touching a non-dedicated local database.

## Phase 2 PR Change Summary

- `users`: documentation corrected to `active` / `disabled` / `suspended`.
- `refresh_tokens`: no schema change; documentation aligned to Alembic names and delete behavior.
- `audit_logs`: added `ix_audit_logs_payload_gin`.
- `idempotency_keys`: documentation corrected to `uq_idempotency_keys_user_id_request_key`; no task `idem_key` is added.
- `tasks`: added nullable `queued_at`, backfilled from `created_at` for queued/running/terminal rows; added range checks and JSONB GIN indexes.
- `task_results`: added numeric/range checks and JSONB GIN indexes.
- `task_assets`: added non-empty role/source_type checks, numeric/range checks, and metadata GIN index; removed old hard enum documentation for role/source_type.
- `task_events`: added payload GIN index; documented `warning` instead of old `warn`.
- `task_usage_records`: added non-negative counters/cost checks and metadata GIN index.
- `image_edits`: documentation replaced old `result_id/task_id/new_result_id/annotation_data` shape with current `source_result_id/edit_task_id/edited_result_id/selection/metadata` shape; added selection and metadata GIN indexes.
