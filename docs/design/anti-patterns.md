# UI 禁止事项与反模式（主图生成工作台）

## 1. 视觉层面反模式

1. **高饱和大面积撞色**：破坏“高级、克制、专业”调性。
2. **过重阴影/毛玻璃滥用**：导致页面“脏”和信息发虚。
3. **同屏多种圆角体系**：8/12/20/30 混用造成风格不统一。
4. **过多渐变并列竞争**：主按钮、标签、卡片同时渐变会失焦。
5. **图标风格混杂**：线性 + 面性 + emoji 混搭。

## 2. 布局层面反模式

1. **模块无分组**：上传、参数、文案混在一个大盒子中。
2. **层级反转**：把次要说明做得比主操作更显眼。
3. **右侧结果区拥挤**：卡片间距过小，信息互相干扰。
4. **导航与内容宽度不一致**：导致页面“断层”。
5. **内容贴边**：顶部导航或页面边缘无留白。

## 3. 交互层面反模式

1. **点击元素无 hover/focus 反馈**。
2. **用颜色作为唯一状态信号**（无文案/图标辅助）。
3. **生成中允许重复提交但无防抖/禁用提示**。
4. **错误提示只在 toast 一闪而过**，无就地信息。
5. **hover 使用 scale 导致卡片抖动和布局跳动**。

## 4. 表单与文案反模式

1. **字段命名不一致**（如“风格”“样式”“氛围”混用）。
2. **占位文案替代标签**（失焦后信息消失）。
3. **中文术语机器翻译感强**（如“创建图片任务”替代“开始生成”）。
4. **参数单位不明确**（质量范围、亮度范围未标注）。
5. **长段说明压在首屏**，抢占用户操作空间。

## 5. 结果展示反模式

1. **不同状态卡片视觉差异过小**，无法快速扫读。
2. **卡片操作图标无说明且顺序不固定**。
3. **缩略图比例混乱**，瀑布流失衡。
4. **失败卡片无重试入口**。
5. **完成态没有“可下载/可预览”的明确反馈**。

## 6. 工程实现反模式（React）

1. **颜色/间距魔法数散落组件**，不走 token。
2. **同一组件在不同页面复制粘贴微调**，不抽象共用版本。
3. **状态 class 命名不一致**（active/selected/current 混用）。
4. **把业务判断写进纯展示组件**，导致复用困难。
5. **先写页面再补规范**，最终出现样式债务。

## 7. 发布前自检清单（必须）

- 是否所有卡片都遵循统一圆角、阴影、边框规则？
- 是否所有按钮都有清晰层级（主/次/轻）？
- 是否所有状态均有“颜色 + 文本”双表达？
- 是否 1440~1600 下主流程一屏可达且不拥挤？
- 是否所有术语为自然中文并与业务一致？
- 是否没有引入 emoji 作为功能图标？
