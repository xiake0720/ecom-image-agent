# 页面布局规则（主图生成工作台优先）

## 1. 目标布局
- 适配桌面端 `1440 / 1680 / 1920 / 4K`，优先保证工作台感和图片浏览效率。
- 页面结构保持：
  1. 顶部导航栏
  2. 左侧操作区
  3. 右侧任务进度与结果区

## 2. 页面容器
- 推荐内容宽度：`width: min(1920px, calc(100vw - 56px))`
- 顶部导航与主内容共用同一宽度基线
- 卡片之间保持 `18px ~ 28px` 的稳定间距

## 3. 左右分栏比例
- 左侧操作区：`clamp(460px, 36vw, 520px)`
- 右侧结果区：使用剩余空间自适应
- 两栏间距：`28px`
- `< 1200px` 时退化为单列，上方操作区、下方结果区

## 4. 左侧操作区顺序
左侧模块顺序不变：
1. 上传商品图
2. 上传参考图
3. 选择平台
4. 选择风格
5. 参数设置
6. 文案备注
7. 主操作按钮

## 5. 上传区布局
- 商品图上传区：
  - 使用更明确的主上传位
  - 主上传位与替换卡片组成双列布局
  - 已上传后保留大预览区，避免操作区显得拥挤
- 参考图上传区：
  - 使用统一尺寸缩略图网格
  - 默认桌面端 3 列，较窄桌面降为 2 列，移动端单列
  - “添加参考图”卡片与已上传缩略图保持同尺寸语言

## 6. 右侧结果区布局
- 顶部固定任务进度卡
- 下方为结果板卡片，内部使用图片网格，不做图文列表
- 结果区响应式规则：
  - `>= 1680px`：4 列
  - `1200px ~ 1679px`：3 列
  - `< 1200px`：2 列
- 结果卡片图片比例统一为 `4:5`
- 卡片高度、操作区和状态标签位置保持一致，便于横向扫读

## 7. 顶部导航约束
- 本轮不重做顶部导航
- 品牌区、一级导航、右上 action group 保持三段式信息结构
- 允许做轻量视觉精修，但不改变导航职责与入口数量
