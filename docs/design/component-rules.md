# 组件规范（主图生成工作台）

## 1. 上传卡片规范

## 1.1 结构

- 卡片标题（如：上传商品图）
- 上传拖拽区（虚线边框）
- 已上传缩略图与文件信息
- 辅助提示（格式、尺寸、大小）

## 1.2 交互

- 默认态：浅底 + 虚线边框
- Hover：边框改为品牌浅色，背景轻微提亮
- 拖入态：边框加深 + 提示文案“松手上传”
- 错误态：红色边框 + 错误原因（文件过大/格式不支持）

## 1.3 规格

- 拖拽区最小高度：`128px`
- 缩略图圆角：`12px`
- 支持格式文案统一：`JPG / PNG / WebP`

---

## 2. 平台选择与风格标签规范

## 2.1 平台选择（单选）

- 使用单选组（Radio Group），不可伪装成普通按钮。
- 选中态：品牌色描边 + 浅品牌底色 + 左侧选中圆点。
- 平台数量较多时支持折行，间距保持 `12px`。

## 2.2 风格标签（多选）

- 使用可切换标签 Chips（带缩略图可选）。
- 选中态：品牌边框 + 背景加深 4~8%。
- 未选中态：中性浅底，文本清晰可读。
- 最多推荐选择数量：`3~5` 个，超限时提示“建议不超过 5 个风格”。

---

## 3. 表单与参数区规范

## 3.1 输入类控件

- Input / Textarea / Select 高度统一：`40px`（textarea 除外）
- 标签与控件间距：`6px`
- 控件间垂直间距：`12px`
- Placeholder 使用 `--color-text-tertiary`

## 3.2 滑杆参数

- 滑杆左侧显示参数名，右侧显示当前值。
- 支持最小/最大值提示（可选）。
- 多滑杆并列时右侧数值位宽保持一致。

## 3.3 文案备注

- Textarea 最小高度：`88px`
- 文案建议提示：如“可填写品牌名、卖点、禁用词”。
- 可选字符计数：`0/200`

---

## 4. 任务进度区规范

## 4.1 阶段进度

- 主进度条：展示整体百分比。
- 阶段节点：至少 4~5 个，名称固定，避免频繁改文案。
- 当前阶段高亮，历史阶段弱化但可见。

## 4.2 状态文案

- 进行中：`正在生成第 x / y 张...`
- 完成：`任务已完成，共生成 y 张`
- 异常：`生成中断，请重试或调整参数`

---

## 5. 结果图卡片规范

## 5.1 卡片结构

- 顶部：图片预览（固定比例）
- 中部：状态标签 + 操作图标（预览/下载/重试）
- 底部：简要信息（风格、场景、生成时间）

## 5.2 图片区

- 建议比例：`4:5`（主图常用）
- 加载中使用骨架屏，不使用闪烁 GIF。
- 图片失败显示统一占位图 + 重试按钮。

## 5.3 操作图标

- 统一使用线性图标（24x24 viewBox）。
- 点击热区不少于 `32x32px`。
- Hover 仅颜色变化，不位移。

---

## 6. 状态标签规范

- `完成`：绿底绿字（success）
- `队列中`：黄底棕字（warning）
- `生成中`：蓝底蓝字（info）
- `失败`：红底红字（error）

规则：
- 标签高度：`24px`
- 内边距：`0 10px`
- 字号：`12px`，`500`
- 文字必须明确，不可只写“处理中”这类模糊词

---

## 7. 按钮规范

## 7.1 层级

- Primary：主操作（开始生成）
- Secondary：次操作（生成详情页方案）
- Tertiary/Text：轻操作（取消、清空）

## 7.2 视觉

- Primary 可用蓝紫渐变（品牌蓝 → 辅助紫），突出主路径。
- Secondary 使用白底描边。
- Disabled 降低对比并禁用 hover。

## 7.3 尺寸

- 默认高度：`44px`
- 圆角：`12px`
- 按钮组间距：`12px`
- 主按钮在左，次按钮在右（与当前工作台习惯一致）

## 7.4 文案

- 主按钮用动词开头：`开始生成`
- 次按钮强调目标：`生成详情页方案`
- Loading 文案：`生成中...`（附小型加载图标）
