# 组件规范（主图生成工作台）

## 1. 上传卡片组件体系

### 1.1 统一原则
- 商品图与参考图复用同一套上传卡片语言：标题、说明、计数、缩略图、添加卡片。
- 不允许空状态和已上传状态混用成两套完全不同的视觉系统。
- 上传卡片 hover 仅做轻量抬升和边框高亮，不做夸张动画。

### 1.2 商品图上传卡片
- 商品图使用主上传位，优先展示单张大预览。
- 已上传后保留大预览卡，同时提供替换入口。
- 主商品图可选支持删除，删除后回到空状态主上传位。
- 商品图缩略图文案至少包含：主商品图标识、文件名。

### 1.3 参考图上传卡片
- 参考图使用统一缩略图网格，不区分首图与普通图尺寸。
- 每张参考图右上角必须提供轻量删除按钮。
- 删除后网格自动补位，不保留空洞。
- 支持追加上传，最多 6 张。
- “添加参考图”卡片必须与参考图缩略图保持统一尺寸语义。

## 2. 结果图卡片

### 2.1 结构
- 上部：图片预览，固定 `4:5` 比例。
- 中部：状态标签 + 操作按钮组。
- 下部：结果标题，简要描述风格与场景。

### 2.2 操作区
- 操作按钮保持右对齐，尺寸统一。
- 小眼睛按钮必须打开当前页图片预览弹层。
- 下载、重试按钮保留原有占位，不要求本轮扩展后台逻辑。

## 3. 图片预览弹层
- 使用当前页轻量 modal，不跳转浏览器新页面。
- 弹层包含：标题区、大图预览区、关闭按钮。
- 关闭方式至少包含：
  - 点击遮罩
  - 点击右上角关闭按钮
  - 按 `Esc`
- 弹层视觉保持简洁、偏 SaaS 工作台风格，使用半透明遮罩和高层级卡片。

## 4. 状态标签
- `完成`：绿色语义
- `队列中`：黄色语义
- `生成中`：蓝色语义
- `失败`：红色语义
- 标签高度统一为 `24px` 左右，避免挤压结果卡片操作区

## 5. 交互细节
- 上传 input 在选择文件后需要重置 value，允许用户重复选择同一文件。
- 本地预览图必须统一回收 object URL，避免重复预览造成内存泄漏。
- 结果卡片 hover 可以轻微抬升，但不能影响网格稳定性。
